⚙ Evento de Leilao

Funcionalidade de execucao do leilao em tempo real com registro de lances e controle de ciclo de vida.

Ciclo de Vida do Leilao

ABERTOrecem-criado
➔
EM_ANDAMENTOevento ativo
➔
FINALIZADOencerrado
Status Descricao Acoes disponiveis
ABERTO Leilao recem-criado, aguardando inicio Botao "Comecar Leilao"
EM_ANDAMENTO Evento ativo, lotes abertos para lances Registrar lances + Botao "Encerrar Leilao"
FINALIZADO Evento encerrado, sem retorno Apenas visualizacao do resultado
O status eh irreversivel. Uma vez finalizado, o leilao nao pode ser reiniciado.

Fluxo Passo a Passo

1. Acessar o Evento

Opcao A — Pela lista de leiloes:

  1. Menu lateral → Leiloes
  2. Na tabela, clique no icone de martelo verde na coluna "Acoes"

Opcao B — Pela visualizacao do leilao:

  1. Menu lateral → Leiloes → clique no icone de olho para ver detalhes
  2. No header, clique no botao verde "Evento"
Ambas as opcoes levam para a rota: /leiloes/:id/evento

2. Iniciar o Leilao

  1. Na tela do evento, o status aparece como ABERTO
  2. Clique no botao verde "Comecar Leilao"
  3. Modal de confirmacao aparece com botao "Iniciar" (verde)
  4. Ao confirmar:
    • Status do leilao muda para EM_ANDAMENTO
    • Lotes nao sao avancados automaticamente — cada lote deve ser avancado manualmente via botao "Avancar Status"
    • Tabela de lotes com input de lance aparece para lotes ja em AGUARDANDO_LANCE
Leilao com status FINALIZADO nao pode ser reiniciado. A transicao de status eh irreversivel.

3. Registrar Lances

  1. Na tabela "Lotes Aguardando Lance", cada lote tem um campo de valor
  2. Digite o valor do lance no campo numerico
  3. Clique no botao verde "Confirmar"
  4. Modal mostra o valor e o codigo do lote para confirmacao
  5. Ao confirmar:
    • precoCompra do lote eh salvo
    • Status do lote muda para FINALIZADO
    • Lote migra da tabela "Aguardando" para "Lotes Arrematados"
    • KPIs e barra de progresso atualizam automaticamente

4. Encerrar o Leilao

  1. Clique no botao vermelho "Encerrar Leilao"
  2. Modal informa quantos lotes ficaram sem lance (se houver)
  3. Ao confirmar:
    • Status do leilao muda para FINALIZADO
    • Lotes sem lance recebem nao_vendido_no_leilao = "S"
    • Botoes "Comecar" e "Encerrar" desaparecem
    • Tela mostra resultado final: "Este leilao foi encerrado"

API Endpoints

PATCH /api/leiloes/{id}/iniciar
Permissao: LEILOES:EDITAR
Pre-condicao: Status do leilao = ABERTO (FINALIZADO nao pode ser reiniciado)
Efeito: Status → EM_ANDAMENTO. Lotes nao sao avancados automaticamente.
PATCH /api/leiloes/{id}/encerrar
Permissao: LEILOES:EDITAR
Pre-condicao: Status do leilao = EM_ANDAMENTO
Efeito: Status → FINALIZADO, lotes sem lance recebem flag nao_vendido_no_leilao = "S"
PATCH /api/lote/{id}/preco 
Body: { "precoCompra": 5500.00 }
Permissao: LOTES:EDITAR
Pre-condicao: Status do lote = AGUARDANDO_LANCE
Efeito: Salva preco, status do lote → FINALIZADO
PATCH /api/lote/{id}/status/avancar
Permissao: LOTES:EDITAR
Efeito: Avanca o lote de AGUARDANDO_ESCRITORIO → AGUARDANDO_LANCE manualmente
GET /api/publico/leilao/{leilaoId}/lotes
Permissao: JWT obrigatorio (qualquer funcionario autenticado)
Efeito: Retorna lotes com status AGUARDANDO_LANCE ou FINALIZADO
PATCH /api/publico/lote/{id}/preco 
Body: { "precoCompra": 5500.00 }
Permissao: JWT obrigatorio (qualquer funcionario autenticado)
Pre-condicao: Lote em AGUARDANDO_LANCE
Efeito: Salva preco, status → FINALIZADO, publica no Redis → WebSocket atualiza todas as telas

Link de Evento para Funcionario

O operador pode gerar um link e enviar para um funcionario externo (ex.: anotador de lances em campo). Esse link abre uma tela dedicada com os lotes em tempo real e permite registrar lances.

O funcionario precisa estar logado no sistema para acessar o link. Caso nao esteja autenticado, sera redirecionado para a tela de login.

Como gerar o link

  1. Com o leilao em EM_ANDAMENTO, clique no botao "Gerar Link" (icone de corrente)
  2. A URL e copiada automaticamente para a area de transferencia
  3. Envie o link para o funcionario (WhatsApp, email, etc.)

Formato do link: {frontendUrl}/#/publico/evento/{leilaoId}
